# MechMind · מהנדס-העל ⚙️

**עומק מכני לפי דרישה — למהנדסי תעשייה וניהול ומנהלי פרויקטים.**

מתארים כוונה בשפה חופשית או מעלים תמונת שרטוט — ומקבלים תוצר הנדסי אמיתי:
מודל STEP/DXF, כתב כמויות Excel, בדיקת שפיות חוזק, המלצת חומר ותהליך ייצור,
ותוכנית פרויקט עם מפרט RFQ לספק.

> **העיקרון המרכזי:** כל מספר הנדסי (מאמץ, מקדם ביטחון, נפח, מסה, עלות) מחושב
> בקוד דטרמיניסטי — לא על-ידי ה-AI. ה-AI מפרש כוונה ומנסח; הכלים מחשבים.

## הרצה מקומית — בדיוק הפקודות האלה

### 1. Backend (טרמינל ראשון)

```bash
cd mechmind
python3.11 -m venv .venv
.venv/bin/pip install -r requirements.txt
cp .env.example .env
# ערוך את .env והדבק את מפתח ה-API שלך:
#   ANTHROPIC_API_KEY=sk-ant-...
.venv/bin/uvicorn backend.main:app --host 0.0.0.0 --port 8000
```

### 2. Frontend (טרמינל שני)

```bash
cd mechmind/frontend
npm install
npm run dev
```

### 3. פתח בדפדפן

```
http://localhost:5174
```

**בלי מפתח API?** המוצר עדיין עובד: בדיקת חוזק, יועץ חומרים ותכנון ייצור הם
דטרמיניסטיים לחלוטין וזמינים דרך ה-API. הצ'אט, קורא השרטוטים וסטודיו ה-CAD
דורשים מפתח (יוצג באנר "AI כבוי" בממשק).

### הרצת הבדיקות (97 בדיקות)

```bash
cd mechmind
.venv/bin/python -m pytest backend/tests -q
```

## קו הבטיחות 🔴

כל תוצר חוזק נושא אוטומטית (נאכף בקוד, לא תלוי במודל):

> אומדן ראשוני בלבד. אינו מחליף חתימת מהנדס רשוי. אין להשתמש בו לתכן סופי
> של ציוד נושא, הרמה, לחץ, או כל דבר שאדם עומד לידו.

- מקדם ביטחון < 2 → דגל אדום + הפניה למהנדס.
- עומס דינמי / עייפות / כלי לחץ → המערכת מסרבת לחשב ומפנה למהנדס.

## אבטחה

המוצר עבר סקירת אבטחה וסקירת קוד אדוורסריות (8 זוויות + סקירת אבטחה ייעודית),
וכל הממצאים תוקנו. עיקרי ההגנות:

- **Sandbox ל-CAD:** סקריפטים שנוצרים ע"י ה-LLM עוברים **ולידציית AST** (allowlist
  ייבוא cadquery/math) ורצים ב-**subprocess מבודד** עם מגבלות CPU/זיכרון/קבצים
  ו-timeout קשיח. הוולידציה חוסמת את כל משפחת רפלקציית הפריים/גנרטור
  (`gi_frame`/`f_back`/`f_builtins`...) — וקטור בריחת ה-RCE הקלאסי — וגם את
  ה-API של קלט/פלט של cadquery (מניעת כתיבת קבצים שרירותית). ב-timeout נהרגת
  כל קבוצת התהליכים (בלי יתומים).
- **בקרת גישה מבוססת אסימונים:** לכל סשן ותוצר יש אסימון בלתי-נחיש
  (`secrets.token_urlsafe`). הורדות ורשימת תוצרים דורשות את האסימון — אי-אפשר
  למנות מזהים סדרתיים ולהוריד תוצרים או להמשיך שיחה של סשן אחר.
- **הגנת path traversal** בהורדות; **HTML escape** + חסימת טעינת משאבים חיצוניים
  (רק `data:`) בייצוא PDF — סוגר SSRF/הזרקה.
- הגבלת קצב מודעת-proxy (`X-Forwarded-For`) לכל נקודות ה-LLM. מפתחות רק ב-`.env`.
- קלט JSON מה-LLM עובר נורמליזציה — פלט פגום לא מפיל את השרת (בלי 500 עם stack trace).

## פריסה לענן

**הדרך הקלה — Render Blueprint (backend + frontend יחד):**
במאגר יש `mechmind/render.yaml`. ב-Render: **New → Blueprint → הצבע על הריפו/הענף**.
זה מקים את שני השירותים אוטומטית. אחר כך הגדר את משתני הסביבה המסומנים
`sync: false`: `ANTHROPIC_API_KEY`, `CORS_ORIGINS` (כתובת ה-frontend),
ו-`VITE_API_URL` (כתובת ה-backend).

**Backend ידני (Docker):** `mechmind/Dockerfile` כולל את כל ספריות המערכת
(pango/cairo ל-WeasyPrint, libGL ל-CadQuery), רץ כמשתמש לא-root, ומאחסן SQLite
בדיסק מתמיד. כל פלטפורמת קונטיינרים תריץ אותו.

**Frontend ידני (Vercel / static):** `cd mechmind/frontend && npm run build`,
פרסם את `dist`, והגדר `VITE_API_URL`.

> הערה: WeasyPrint ו-CadQuery דורשים ספריות מערכת native — לכן backend נפרס
> כ-Docker (ה-Dockerfile מטפל בכול), לא כ-runtime Python חשוף.
